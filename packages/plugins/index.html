Plugins

14 plugins

Plugins extend a Claude Code agent with context handlers, tool integrations, and workflow extensions. The packages on this page wrap each plugin as a Flox-installable, Nix-pinned bundle so a team gets identical plugin behavior across every developer's machine and every CI runner. Add a plugin via flox/claude-code-plugin-<name> in any manifest's [install] block.

Why install plugins this way? Claude Code plugins ship as static files — settings, MCP server definitions, command and skill manifests, and instruction fragments. The hard part is not installing them once but keeping them pinned and consistent across a team: which version, which configuration, which set of plugins on which env. Wrapping each plugin as a Flox package solves that. The package declares its contents, the version is locked in your manifest, and activating the Flox env materializes the plugin into the agent's config tree. No manual file copying, no drift between laptops, no "works on mine" when a CI job runs the same agent on a different commit.

All plugins listed below combine with flox/claude — the env auto-assembles plugin fragments under ~/.claude/ so installing a plugin package is the entire setup.
