Skills

10 skills

Skills are domain-specific capability bundles for Claude Code — packaged prompts, scripts, and configurations that give an agent specific expertise. The packages on this page wrap each skill as a Flox-installable, Nix-pinned bundle so a team gets identical agent behavior across every developer's machine. Add a skill via flox/skill-<name> in any manifest's [install] block.

Skills differ from plugins in scope: a plugin extends the agent runtime (MCP servers, command palettes, IDE hooks) while a skill is a tightly-scoped knowledge or workflow bundle. A skill might teach the agent how to write release notes for your team's specific format, how to triage incidents using your runbook, or how to apply your company's house style to documentation. Packaging skills as Flox bundles is the same trick as plugins — version-locked, manifest-included, reproducible. Bring up a fresh laptop, run flox activate, and the same agent with the same skills is ready to go.

All skills listed below pair with flox/claude, which materializes skill manifests into the agent's config tree on activation. No copy-pasting prompts between machines, no drift between a developer's local skills and the CI runner's.
