Rust

Rust + Cargo, stable channel, ready to `cargo run`.

The flox/rust environment packages Rust for one-line inclusion in any Flox manifest. Add it via { remote = "flox/rust" } and the toolchain is pinned, services are wired, and the demo recipe starts cleanly. Published by Flox under MIT OR Apache-2.0 license, upstream Rust, maintained by @rok.

  • Latest stable rustc, cargo, clippy, rustfmt via Flox
  • Platform linker (clang/gcc) included automatically
  • Standard library source for IDE support
  • All toolchain packages pinned to a compatible revision
  • Include from any manifest with one line

Rust Demo

Full Rust development environment with editor tooling and a sample application. Built on top of the minimal rust environment via [include].

Quick start

flox activate
cargo run

Included tools

From the base rust environment (via [include]):

  • rustc — compiler
  • cargo — package manager and build tool
  • clippy — linter
  • rustfmt — code formatter
  • rust-lib-src — standard library source

Added by this environment:

  • rust-analyzer — language server (matched to the compiler version via the rust-toolchain pkg-group)
  • cargo-nextest — fast test runner
  • cargo-watch — file watcher for iterative development
  • cargo-semver-checks — semver compatibility linter

Development workflow

Format the code:

cargo fmt

Run the linter:

cargo clippy -- -D warnings

Build and run:

cargo run

Run tests:

cargo test

Or use the included test script to run all checks:

./test.sh

Documentation

Rust

Minimal Rust toolchain environment designed for use as an [include] base in other Flox environments.

What it provides

  • rustc — the Rust compiler
  • cargo — package manager and build tool
  • clippy — linter
  • rustfmt — code formatter
  • rust-lib-src — standard library source for IDE support
  • clang (macOS) / gcc (Linux) — platform linker

Usage

Include this environment in your own manifest:

[include]
environments = [{ remote = "flox/rust" }]

Then add project-specific packages (rust-analyzer, cargo tools, etc.) in your local manifest.

Package groups

All toolchain packages belong to the rust-toolchain pkg-group. This ensures that rustc, cargo, clippy, and rustfmt are resolved to compatible versions from the same Nixpkgs revision.

When adding packages that must match the compiler version (e.g. rust-analyzer), place them in the same group:

rust-analyzer.pkg-path = "rust-analyzer"
rust-analyzer.pkg-group = "rust-toolchain"

Platform-specific notes

PlatformLinkerExtra libraries
macOSclanglibiconv
Linuxgcc(none)

These are included automatically and scoped to the correct systems via pkg.systems.

See also

For a full development environment with rust-analyzer, cargo tools, and a sample application, see rust-demo.
