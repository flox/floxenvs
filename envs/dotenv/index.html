dotenv

Auto-load `.env` files on Flox activation — zero config, override the path with one variable.

  • Sources .env on every `flox activate`, no extra tools
  • All variables exported automatically via allexport
  • Override the file path with `DOTENV_FILE` in [vars]
  • Silently skipped if the file does not exist
  • Include from any manifest with one line

dotenv-demo
  • Creates a sample .env and activates with it loaded
  • Shows which variables were loaded on activation
  • Override path: `DOTENV_FILE=.env.local` in [vars]
composed manifest.toml 
schema-version = "1.10.0"

[include]
environments = [
  { dir = "../dotenv" }
]

[install]
gum.pkg-path = "gum"
gum.pkg-group = "demo-tools"

[hook]
on-activate = '''
if [[ "${FLOX_ENVS_TESTING:-}" != "1" ]]; then
  if [ -f "$DOTENV_FILE" ]; then
    LOADED="$(grep -c '=' "$DOTENV_FILE" 2>/dev/null || echo 0)"
    gum style --border double --margin '1 2' \
      --padding '1 4' \
      'dotenv environment ready!' '' \
      "Loaded $LOADED variables from $DOTENV_FILE"
  else
    gum style --border double --margin '1 2' \
      --padding '1 4' \
      'dotenv environment ready!' '' \
      "No $DOTENV_FILE file found." \
      'Create one to auto-load variables.'
  fi
fi
'''

dotenv-demo

A demonstration environment showing how the dotenv base layer automatically loads variables from a .env file.

What it does

On activation this environment:

  1. Includes the minimal dotenv layer which sources .env.
  2. Displays a welcome banner (using gum) showing how many variables were loaded.

Quick start

flox activate -d dotenv-demo

The sample .env ships with the demo so variables are available immediately.

Customizing the file path

Override DOTENV_FILE in your own manifest to load a different file:

[vars]
DOTENV_FILE = ".env.local"

Documentation

dotenv

Automatically load environment variables from a .env file on Flox activation.

Usage

Include this environment in your own manifest:

[include]
environments = ["flox/dotenv"]

By default it loads .env in the current directory. Override the file path with:

[vars]
DOTENV_FILE = ".env.local"

How it works

The on-activate hook uses set -o allexport and source to export every variable defined in the file. If the file does not exist the hook is silently skipped.

Caveats

  • The target file must exist at activation time; it is not an error if it is missing (the hook simply does nothing).
  • Multiline values are not reliably supported with bare source. Use single-line values or a dedicated parser.
  • Comments and inline comments follow standard shell rules.

See also

  • dotenv-demo shows a full working example with a welcome banner and sample variables.
