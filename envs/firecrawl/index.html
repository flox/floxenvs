Firecrawl

Official Firecrawl CLI plus a Claude Code skill bundle for scraping, crawling, searching, and extracting web data.

The flox/firecrawl environment packages firecrawl-cli for one-line inclusion in any Flox manifest. Add it via { remote = "flox/firecrawl" } and the toolchain is pinned, services are wired, and the demo recipe starts cleanly. Published by Flox under ISC license, upstream firecrawl-cli, maintained by @rok.

  • One-line include — firecrawl CLI and skills ready in any env
  • Scrape any URL to clean, LLM-ready markdown
  • Crawl whole sites, map links, and search the web from the shell
  • Claude Code / OpenCode skills drive the CLI from an agent
  • Authenticates via FIRECRAWL_API_KEY or `firecrawl login`

firecrawl-demo

firecrawl-demo
  • Interactive first-run prompt for your firecrawl.dev API key
  • Key persisted to the env cache and reused on later activations
  • Styled banner with quick-start scrape, search, and status commands
composed manifest.toml 
schema-version = "1.11.0"

[include]
environments = [
  { dir = "../firecrawl" }
]

# gum is a demo-only tool for styled terminal output and
# the interactive API-key prompt.

[install]
gum.pkg-path = "gum"
gum.pkg-group = "demo-tools"

[hook]
on-activate = '''
if [[ "${FLOX_ENVS_TESTING:-}" != "1" ]]; then
  dir="${FIRECRAWL_DATA_DIR:-$FLOX_ENV_CACHE/firecrawl}"
  mkdir -p "$dir"
  key_file="$dir/api-key"

  # Ask for the Firecrawl API key on first run. Only prompt
  # when running interactively and the key isn't already set.
  if [[ -z "${FIRECRAWL_API_KEY:-}" && -t 0 ]]; then
    gum style --border double --margin '1 2' --padding '1 4' \
      'Firecrawl demo — scrape, crawl, and search the web' '' \
      'A Firecrawl API key is required.' \
      'Grab one (free tier available) at:' \
      '  https://firecrawl.dev/app/api-keys'
    key="$(gum input --password \
      --placeholder 'fc-...' \
      --prompt 'FIRECRAWL_API_KEY > ')"
    if [[ -n "$key" ]]; then
      printf '%s' "$key" > "$key_file"
      chmod 600 "$key_file"
      FIRECRAWL_API_KEY="$key"
      export FIRECRAWL_API_KEY
      gum style --foreground 42 'Key saved to the env cache. ✓'
    else
      gum style --foreground 214 \
        'No key entered — set FIRECRAWL_API_KEY later or run `firecrawl login`.'
    fi
  fi

  gum style --border double --margin '1 2' --padding '1 4' \
    'Firecrawl CLI' '' \
    'Check auth + credits:' \
    '  firecrawl --status' '' \
    'Scrape a page to clean markdown:' \
    '  firecrawl scrape "https://firecrawl.dev" -o page.md' '' \
    'Search the web:' \
    '  firecrawl search "flox dev environments"' '' \
    'Drive it from Claude Code / OpenCode via the bundled skill.'
fi
'''

firecrawl-demo

Interactive walkthrough for the flox/firecrawl environment. On first activation it prompts for your Firecrawl API key, saves it to the env cache, and prints quick-start commands.

Quick start

flox activate -r flox/firecrawl-demo

On activation you’ll be asked for a Firecrawl API key. Get one (free tier available) at https://firecrawl.dev/app/api-keys. The key is stored in $FLOX_ENV_CACHE/firecrawl/api-key and reused on later activations, so you’re only prompted once.

Three things to try

  1. Check auth and remaining credits:

    firecrawl --status

  2. Scrape a page to clean, LLM-ready markdown:

    firecrawl scrape "https://firecrawl.dev" -o page.md

  3. Search the web from the shell:

    firecrawl search "flox dev environments"

Run firecrawl --help or firecrawl <command> --help for the full option set.

Driving it from an agent

The bundled skill lets Claude Code or OpenCode call firecrawl for you — just ask it to “search the web for …” or “scrape this page …” and it picks the right command. The skill files come from the included flox/firecrawl environment.

See also

For the minimal environment without the prompt or banner, see firecrawl.

firecrawl

Minimal Firecrawl environment. Provides the official Firecrawl CLI (firecrawl) for scraping, crawling, searching, and extracting web data, plus a Claude Code / OpenCode skill bundle for driving it from an agent. It includes the flox/claude managed Claude Code config, so the bundled skills are auto-discovered out of the box.

Quick start

Activate directly:

flox activate -r flox/firecrawl
export FIRECRAWL_API_KEY=fc-...   # from https://firecrawl.dev
firecrawl --status
firecrawl scrape "https://firecrawl.dev" -o page.md

Include in your project

Add to your manifest.toml:

[include]
environments = [{ remote = "flox/firecrawl" }]

What is included

PackageDescription
firecrawl-cliOfficial Firecrawl CLI (firecrawl)
skills-firecrawl-cliClaude Code / OpenCode skill bundle
flox/claudeManaged Claude Code config (via [include])

Authentication

Firecrawl needs an API key. Get one (free tier available) at https://firecrawl.dev/app/api-keys, then either:

  • Export it: export FIRECRAWL_API_KEY=fc-...
  • Or run the built-in login: firecrawl login

The key can also be persisted across activations by writing it to $FLOX_ENV_CACHE/firecrawl/api-key; the hook loads it automatically when FIRECRAWL_API_KEY isn’t already set. The firecrawl-demo prompts for the key interactively and saves it there.

Configuration variables

VariableDefaultDescription
FIRECRAWL_API_KEY(empty)Firecrawl API key
FIRECRAWL_DATA_DIR$FLOX_ENV_CACHE/firecrawlState directory

Override either in your own manifest before [include]-ing this environment.

Skill auto-pickup

This env includes flox/claude, so the bundled skills under $FLOX_ENV/share/claude-code/skills/ are discovered by flox-ai and surfaced to Claude Code automatically. The same files are installed for OpenCode under $FLOX_ENV/share/opencode/skills/. The bundle covers search, scrape, crawl, map, interact, parse, download, and monitor.

See also

For an interactive walkthrough that prompts for your API key, see firecrawl-demo.
