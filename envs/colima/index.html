Colima

Colima container runtime ready to include — Docker- compatible containers on macOS and Linux via Lima.

  • Colima + Docker CLI via Flox, no Docker Desktop required
  • Runs as a Flox service with `--start-services`
  • Works on macOS (QEMU/Virtualization.framework) and Linux
  • Include from any manifest with one line

Quick start

colima-demo
  • Launches Colima with `flox services start`
  • Adds docker-compose, lazydocker, oxker, ctop, dive
  • Styled welcome with connection and usage hints
composed manifest.toml 
schema-version = "1.10.0"
[include]
environments = [
  { dir = "../colima" }
]

# ============================================================
# Packages
# ============================================================
#
# Container management and inspection tools are grouped
# separately from the base colima/docker packages defined
# in the included colima environment.

[install]

# UI and demo tools
gum.pkg-path = "gum"
gum.pkg-group = "demo-tools"

# Docker Compose for multi-container workflows
docker-compose.pkg-path = "docker-compose"
docker-compose.pkg-group = "demo-tools"

# Container TUI tools
lazydocker.pkg-path = "lazydocker"
lazydocker.pkg-group = "demo-tools"

oxker.pkg-path = "oxker"
oxker.pkg-group = "demo-tools"

ctop.pkg-path = "ctop"
ctop.pkg-group = "demo-tools"

# Image layer inspector
dive.pkg-path = "dive"
dive.pkg-group = "demo-tools"


# ============================================================
# Activation hook
# ============================================================
#
# The colima service setup comes from the included colima
# environment. This hook adds demo-specific connection info.

[hook]
on-activate = '''

if [[ "${FLOX_ENVS_TESTING:-}" != "1" ]]; then
  gum style --border double --margin '1 2' --padding '1 4' \
    'Start Colima in the background:' \
    '  flox services start' \
    '  flox activate --start-services' '' \
    'Once running, try:' \
    '  docker run hello-world' \
    '  docker-compose up' '' \
    'Container management TUIs:' \
    '  lazydocker    — full container/image/volume dashboard' \
    '  oxker         — lightweight container viewer' \
    '  ctop          — top-like container metrics' \
    '  dive <image>  — explore image layers'
fi

'''

[options]
systems = ["aarch64-darwin", "aarch64-linux", "x86_64-linux"]

Documentation

colima

Minimal Colima environment. Include it in your own manifest to get a working container runtime with Docker CLI.

Quick start

Activate directly:

flox activate -r flox/colima --start-services

Include in your project

Add to your manifest.toml:

[include]
environments = ["flox/colima"]

Defaults

SettingValue
RuntimeColima (Lima-based containers)
CLIDocker client
VM backendDefault (QEMU or Virtualization.framework)

Usage

Once the service is running:

docker run hello-world
docker ps
docker images

Customizing

Override settings in your own manifest as needed. For example, to add docker-compose:

[install]
docker-compose.pkg-path = "docker-compose"

See also

For a full setup with container management TUIs (lazydocker, oxker, ctop, dive), see colima-demo.
