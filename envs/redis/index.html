Redis

Redis ready to include — in-memory key/value store with sane defaults, runs as a Flox service on port 16379.

The flox/redis environment packages Redis for one-line inclusion in any Flox manifest. Add it via { remote = "flox/redis" } and the toolchain is pinned, services are wired, and the demo recipe starts cleanly. Published by Flox under BSD-3-Clause license, upstream Redis, maintained by @rok.

  • Latest stable Redis server and CLI via Flox
  • Binds to 127.0.0.1:16379 by default (override via vars)
  • Data persisted under `$FLOX_ENV_CACHE/redis`
  • Boots as a Flox service with `--start-services`
  • Override `REDIS_HOST` and `REDIS_PORT` in your manifest
New to Flox? Install it in 30 seconds: curl -L https://downloads.flox.dev/by-env/stable/install.sh | bash — then run the activate command below. Docs →

Recipes

Quick start

redis-demo
  • Launches Redis as a Flox service on port 16379
  • Connects via `redis-cli` with no extra configuration
  • SET/GET examples with styled terminal output
composed manifest.toml 
schema-version = "1.10.0"

[include]
environments = [
  { dir = "../redis" }
]

# gum is a demo-only tool for styled terminal output.

[install]
gum.pkg-path = "gum"
gum.pkg-group = "demo-tools"

[hook]
on-activate = '''
if [[ "${FLOX_ENVS_TESTING:-}" != "1" ]]; then
  gum style --border double --margin '1 2' \
    --padding '1 4' \
    'Start Redis in the background:' \
    '  flox services start' \
    '  flox activate --start-services' '' \
    'Connect to Redis:' \
    "  redis-cli -h $REDIS_HOST -p $REDIS_PORT" '' \
    'Connection information:' \
    "  host=$REDIS_HOST" \
    "  port=$REDIS_PORT"
fi
'''

redis-demo

Full Redis demo environment that walks you through connection, basic operations, and service management with Flox.

For a minimal environment to include in your own project, see redis or use flox/redis on FloxHub.

Quick start

flox activate -r flox/redis-demo --start-services

Then connect:

redis-cli -h "$REDIS_HOST" -p "$REDIS_PORT"

What this demo includes

  • Redis server and CLI (latest stable)
  • gum for styled terminal output
  • Service definition for background redis-server
  • Connection info display on activation

SET/GET walkthrough

After starting the service, try basic operations:

redis-cli -h "$REDIS_HOST" -p "$REDIS_PORT"
127.0.0.1:16379> SET greeting "hello"
OK
127.0.0.1:16379> GET greeting
"hello"
127.0.0.1:16379> DEL greeting
(integer) 1
127.0.0.1:16379> QUIT

Or run commands directly from the shell:

redis-cli -h "$REDIS_HOST" -p "$REDIS_PORT" \
  SET mykey "myvalue"
redis-cli -h "$REDIS_HOST" -p "$REDIS_PORT" \
  GET mykey

Customizing

Override these vars in your own manifest:

[vars]
REDIS_HOST = "0.0.0.0"   # listen on all interfaces
REDIS_PORT = "6379"       # use standard port

Data directory

Redis data is stored in $FLOX_ENV_CACHE/redis. This persists across activations. To start fresh:

rm -rf "$FLOX_ENV_CACHE/redis"

Using the minimal version

If you don’t need the demo experience, include the minimal environment in your own manifest:

[include]
environments = [{ remote = "flox/redis" }]

This gives you Redis with sane defaults. Override vars in your own manifest as needed.

Service management

flox services start        # start redis
flox services stop         # stop redis
flox services status       # check status
flox services logs redis   # view logs

Documentation

redis

Minimal Redis environment. Include it in your own manifest to get a working Redis server and CLI with sane defaults.

Quick start

Activate directly:

flox activate -r flox/redis --start-services

Include in your project

Add to your manifest.toml:

[include]
environments = [{ remote = "flox/redis" }]

Then customize vars in your own manifest to override defaults.

Defaults

SettingValue
VersionLatest stable (tracks redis)
Host127.0.0.1
Port16379
Data dir$FLOX_ENV_CACHE/redis

Customizing

Override these vars in your own manifest:

[vars]
REDIS_HOST = "0.0.0.0"   # listen on all interfaces
REDIS_PORT = "6379"       # use standard port

Environment variables

After activation, these are available in your shell:

VariableDescription
REDIS_HOSTBind address for the server
REDIS_PORTPort the server listens on
REDIS_DATAPath to the data directory

Connection string

Most Redis clients accept a URL in this form:

redis://$REDIS_HOST:$REDIS_PORT

Or connect with the CLI:

redis-cli -h "$REDIS_HOST" -p "$REDIS_PORT"

See also

For a full walkthrough with SET/GET examples and styled terminal output, see redis-demo.
