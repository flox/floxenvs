Podman

Podman ready to include — rootless, daemonless OCI containers on macOS and Linux.

  • Podman + podman-compose via Flox, no daemon required
  • QEMU VM included on macOS for Linux containers
  • Containers policy auto-created on Linux for rootless use
  • Docker-compose compatible with `podman-compose`
  • Include from any manifest with one line

Quick start

podman-demo
  • Adds podman-tui, undocker, and gum for interactive use
  • `podman run <image>` or `podman-compose up` right away
  • VM auto-stopped on shell exit (macOS)
composed manifest.toml 
schema-version = "1.10.0"

[include]
environments = [
  { dir = "../podman" }
]

[install]
gum.pkg-path = "gum"
gum.pkg-group = "demo-tools"
undocker.pkg-path = "undocker"
undocker.pkg-group = "demo-tools"
podman-tui.pkg-path = "podman-tui"
podman-tui.pkg-group = "demo-tools"

[hook]
on-activate = '''
if [[ "${FLOX_ENVS_TESTING:-}" != "1" ]]; then
  gum style --border double --margin '1 2' \
    --padding '1 4' \
    'Podman environment ready!' '' \
    'podman version:' \
    "  $(podman --version)" '' \
    'Commands:' \
    '  podman run <image>     Run a container' \
    '  podman-compose up      Start services' \
    '  podman-tui             Terminal UI' \
    '  undocker <image>       Extract image'
fi
'''

[profile]
common = '''
if [ "$(uname -s)" = 'Darwin' ] \
    && [[ "${FLOX_ENVS_TESTING:-}" != "1" ]]; then
  trap 'gum confirm "Stop virtual machine?" \
    && gum spin --spinner dot \
         --title "Stopping machine..." \
         -- podman machine stop \
    ; echo "Podman machine stopped"' EXIT
fi
'''

Podman Demo

Interactive demo environment built on the minimal podman/ base layer.

Usage

cd podman-demo
flox activate

What’s included

Everything from the base podman environment, plus:

  • gum — interactive prompts and styled output
  • undocker — extract container images to a directory
  • podman-tui — terminal UI for managing containers

Behaviour

Welcome banner

On activation a styled banner is printed showing the podman version and available commands. The banner is suppressed when FLOX_ENVS_TESTING=1.

macOS VM stop on exit

When running on macOS, exiting the shell triggers a confirmation prompt asking whether to stop the Podman virtual machine. This is also suppressed during testing.

Minimal environment

For a lighter setup without the demo tools, use the podman/ environment directly.

Documentation

Podman

Minimal Podman environment providing rootless containers.

Usage

Include this environment in your own manifest:

[include]
environments = ["flox/podman"]

What’s included

  • podman — rootless container engine
  • podman-compose — docker-compose compatible tool
  • qemu — virtual machine support (macOS only)

Notes

macOS

Podman on macOS needs a Linux virtual machine. After activating the environment, run:

podman machine init
podman machine start

Linux

A containers policy.json is created automatically in ~/.config/containers/ on first activation if one does not already exist.

docker-compose compatibility

Use podman-compose as a drop-in replacement for docker-compose. It reads the same compose.yaml files.

Demo

See podman-demo/ for an interactive demo environment with additional tools.
