Playwright

Playwright ready to include — browser automation, E2E tests, MCP server, and Claude Code skill in one env.

The flox/playwright environment packages Playwright for one-line inclusion in any Flox manifest. Add it via { remote = "flox/playwright" } and the toolchain is pinned, services are wired, and the demo recipe starts cleanly. Published by Flox under Apache-2.0 license, upstream Playwright, maintained by @rok.

  • Playwright + test runner + browser drivers via Flox
  • Microsoft playwright-cli for record/replay workflows
  • Playwright MCP server runs as an opt-in Flox service
  • Claude Code / OpenCode skill auto-discovered on activation
  • Include from any manifest with one line

playwright-demo

  • Starts the Playwright MCP server for agent-driven testing
  • `playwright-cli open <url>` for interactive record/replay
composed manifest.toml 
schema-version = "1.11.0"

[include]
environments = [
  { dir = "../playwright" }
]

# gum is a demo-only tool for styled terminal output.

[install]
gum.pkg-path = "gum"
gum.pkg-group = "demo-tools"

[hook]
on-activate = '''
if [[ "${FLOX_ENVS_TESTING:-}" != "1" ]]; then
  gum style --border double --margin '1 2' \
    --padding '1 4' \
    'Playwright demo — browser automation in a Flox env' '' \
    'Install the chromium binary (first time only):' \
    '  playwright install chromium' '' \
    'Run the sample test against demo.playwright.dev:' \
    '  playwright test' '' \
    'Or expose Playwright as an MCP service:' \
    '  flox services start playwright-mcp' '' \
    "MCP endpoint: http://${PLAYWRIGHT_MCP_HOST}:${PLAYWRIGHT_MCP_PORT}/" '' \
    'Or drive a browser directly from the CLI:' \
    '  playwright-cli open https://playwright.dev'
fi
'''

playwright-demo

Interactive walkthrough for the flox/playwright environment. Ships a tiny sample test that exercises demo.playwright.dev/todomvc so you can verify the toolchain end-to-end with one command.

Quick start

flox activate -r flox/playwright-demo
playwright install chromium
playwright test

The activation banner prints the commands above plus links to the MCP service and playwright-cli.

What’s inside

FilePurpose
tests/example.spec.tstodomvc smoke test
playwright.config.tsOne chromium project, headless

PLAYWRIGHT_HEADLESS=false playwright test --headed opens a real browser window.

Three things to try

  1. Run the sample test:

    playwright test

  2. Drive a browser interactively from the CLI:

    playwright-cli open https://playwright.dev --headed
playwright-cli type "page object"
playwright-cli press Enter

  3. Expose Playwright over MCP:

    flox services start playwright-mcp

    Point Claude Desktop, Cursor, or VS Code’s MCP client at http://$PLAYWRIGHT_MCP_HOST:$PLAYWRIGHT_MCP_PORT/.

Documentation

playwright

Minimal Playwright environment. Provides the Playwright library and test runner, browser drivers, Microsoft’s playwright-cli, the Playwright MCP server, and a Claude Code / OpenCode skill bundle for driving playwright-cli from an agent.

Quick start

Activate directly:

flox activate -r flox/playwright
playwright install chromium   # one-time browser fetch
playwright test               # run tests in $PWD

Include in your project

Add to your manifest.toml:

[include]
environments = [{ remote = "flox/playwright" }]

What is included

PackageDescription
playwrightJS framework + playwright CLI
playwright-test@playwright/test runner
playwright-driverBrowser drivers (chromium, firefox, webkit)
playwright-cliMicrosoft playwright-cli — record / replay
playwright-mcpPlaywright MCP server (STDIO + HTTP)
skills-playwright-cliClaude Code / OpenCode skill for playwright-cli

Configuration variables

VariableDefaultDescription
PLAYWRIGHT_MCP_HOST127.0.0.1MCP server bind address
PLAYWRIGHT_MCP_PORT18931MCP server port (ADR-003 prefix)
PLAYWRIGHT_HEADLESStrueDefault headless flag
PLAYWRIGHT_BROWSERS_PATH$PLAYWRIGHT_DATA_DIR/browsersBrowser cache
PLAYWRIGHT_DATA_DIR$FLOX_ENV_CACHE/playwrightState directory

Override any in your own manifest before [include]-ing this environment.

Services

The MCP server runs as an opt-in background service:

flox services start playwright-mcp
flox services status
flox services logs playwright-mcp
flox services stop playwright-mcp

Point Claude Desktop, Cursor, or any MCP-aware client at http://${PLAYWRIGHT_MCP_HOST}:${PLAYWRIGHT_MCP_PORT}/.

The service starts a headless Chromium per session. Run playwright install chromium once before starting the service if the browser binary isn’t cached yet.

Skill auto-pickup

When activated alongside flox/claude, the bundled SKILL.md at $FLOX_ENV/share/claude-code/skills/playwright-cli/ is discovered by claude-managed and surfaced to Claude Code. Same for OpenCode under $FLOX_ENV/share/opencode/skills/playwright-cli/.

For an interactive walkthrough with a sample test, see playwright-demo.
