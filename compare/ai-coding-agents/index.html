AI coding agents on Flox — compared

This is the canonical 2026 side-by-side comparison for picking an AI coding agent from the Flox catalog. Six terminal-first agents are covered — Anthropic's Claude Code, OpenAI's Codex, Charmbracelet's Crush, Opencode, Openclaw, and the Agent Deck orchestrator — across license, supported provider, multi-model and offline capability, terminal vs IDE surface, and last-update freshness. "Terminal-first" means each agent lives in any shell rather than embedding inside a specific editor like Cursor, Copilot, or Windsurf — the trade-off is no inline IDE UI, the win is editor-agnostic workflows, scriptability, and zero vendor lock-in. Every agent below ships as a reproducible Flox package: one manifest line to install, zero version drift, the identical binary on every developer's machine and every CI runner, on macOS and Linux.

Decision guide

Common questions answered — which agent fits your workflow?

How do these agents compare to Cursor, GitHub Copilot, and Windsurf?

Cursor, Copilot, and Windsurf are IDE-embedded — they live inside a specific editor and assume you stay inside it. Every agent listed here is terminal-first: they run in any shell, work alongside any editor, and are reproducibly pinned by Flox so the binary you got today is the binary you get on every machine tomorrow. The trade-off is no inline UI in the editor; the win is editor-agnostic workflows, scripting, CI, and no vendor lock-in to a single IDE.

Which is the best AI coding agent for Claude users?

Claude Code is the canonical choice for Claude users. It is Anthropic's official CLI, ships with MCP tool support and a bubblewrap sandbox on Linux, and is pre-packaged by Flox so the auto-updater is disabled for reproducible Nix builds. Install it in one line: flox install flox/claude-code.

Which AI coding agent works offline with local models?

Opencode and Openclaw both support local inference. Opencode integrates with Ollama out of the box, and Openclaw is fully self-hosted so all requests stay on your machine. Both are packaged for Flox with a single pkg-path install.

Which AI coding agents are free and open source?

Codex (Apache-2.0), Opencode (MIT), and Openclaw (MIT) are fully open-source. Agent Deck is also MIT. Crush is FSL-1.1-MIT, which is source-available but restricts commercial redistribution for two years. Claude Code is proprietary (Unfree license).

Which AI coding agent supports multi-agent orchestration?

Agent Deck is purpose-built as an orchestration layer: it provides a single TUI for launching, monitoring, and managing sessions across Claude Code, Codex, Crush, Opencode, and Openclaw simultaneously. It uses a plugin architecture for extensible workflows.

Which AI coding agents integrate with my IDE?

All six agents are terminal-first and work alongside any IDE via a side-by-side terminal. Claude Code additionally supports VS Code and JetBrains extensions. If you need deep IDE embedding, pair Claude Code with your editor plugin; for everything else a split-terminal workflow works well.

Which AI coding agent has the best terminal UX?

Crush (by Charmbracelet) and Agent Deck both offer rich TUI interfaces with color, layout, and keyboard navigation. Opencode adds fzf-powered fuzzy search and ripgrep integration. Claude Code and Codex are more minimal but highly efficient for scripting and automation.

Install all of them at once

Flox lets you mix and match agents without conflicts. Add this [install] block to any manifest and every agent is pinned and ready to go: 

# .flox/env/manifest.toml
[install]
claude-code.pkg-path  = "flox/claude-code"
claude-code.publisher = "flox"

codex.pkg-path        = "flox/codex"
codex.publisher       = "flox"

crush.pkg-path        = "flox/crush"
crush.publisher       = "flox"

opencode.pkg-path     = "flox/opencode"
opencode.publisher    = "flox"

openclaw.pkg-path     = "flox/openclaw"
openclaw.publisher    = "flox"

agent-deck.pkg-path   = "flox/agent-deck"
agent-deck.publisher  = "flox"
